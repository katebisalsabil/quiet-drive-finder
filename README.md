# Quiet Drive Finder

Quiet Drive Finder is a beginner-friendly personal web app for finding peaceful, less busy round-trip driving routes near home.

## Project purpose

This app is a simple starting point for a route planning tool that will eventually help you choose quiet driving loops. In Phase 1, the focus is only on project setup, documentation, and making sure everything runs cleanly.

## Tech stack

- React
- Vite
- JavaScript
- ESLint

## Phase plan summary

- Phase 1: Setup, documentation, and confirm the app launches.
- Phase 2: Build the user interface for entering drive preferences and showing route ideas.
- Phase 3: Add route generation and quiet-route selection logic.
- Phase 4: Integrate maps and traffic/location APIs.

## How to install dependencies

1. Open a terminal in the project folder.
2. Run:

```bash
npm install
```

## How to run locally

Start the development server with:

```bash
npm run dev
```

Then open the URL shown in the terminal, usually:

```bash
http://127.0.0.1:4173/
```

## Git workflow

A simple Git workflow for this project is:

1. Create a branch for your work:

```bash
git checkout -b phase-1-setup
```

2. Add changes:

```bash
git add .
```

3. Commit with a clear message:

```bash
git commit -m "Phase 1: setup project and update docs"
```

4. Push your branch:

```bash
git push -u origin phase-1-setup
```

5. Open a pull request when you are ready.

## Future API note

This phase does not use any external APIs. In later phases, the app may connect to traffic or location APIs to make quiet-route suggestions.

## Folder structure

- `src/` contains the React application code.
- `public/` contains static files.
- `index.html` and `vite.config.js` are required Vite files and should be kept.
- `node_modules/` is generated by `npm install` and should not be committed.
- `dist/` is the build output folder and should not be committed.
